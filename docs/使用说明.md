# 使用说明

在服务器上安装、配置并运行 **caddy-alidns**（与如何编译无关）。

---

## 1. 适用场景

| 场景 | 说明 |
|------|------|
| 域名在阿里云 DNS | 自动添加 `_acme-challenge` TXT 完成证书验证 |
| 无法开放 80 端口 | 使用 DNS-01，不依赖 HTTP 验证 |
| 需要 `*.域名` 证书 | 在全局块配置 `acme_dns alidns` |

### 为什么需要「带 alidns 的 Caddy」

官方 Caddy **默认不包含** 阿里云 DNS 质询，申请证书（尤其 **DNS-01 / 通配符**）必须额外集成插件 [**caddy-dns/alidns**](https://github.com/caddy-dns/alidns)。常见做法有两种：

| 方式 | 说明 |
|------|------|
| **Docker 镜像** | 例如社区项目 [Xm798/docker-caddy](https://github.com/Xm798/docker-caddy)，已内置 cloudflare / alidns / tencentcloud 等模块，适合习惯 `docker compose` 部署的场景。 |
| **本仓库（caddy-alidns）** | 提供 **预编译二进制** 或本机 xcaddy 编译，用 **systemd** 跑在宿主机，镜像体积与编排成本更小。 |

二者 Caddyfile 里 `acme_dns alidns { ... }` 写法一致；差别主要在安装与运维方式。若你已在用 Xm798 镜像，**不必**再换本仓库，除非想改为裸机部署或统一用 [youfun/caddy-alidns Releases](https://github.com/youfun/caddy-alidns/releases) 自建镜像中的二进制。

---

## 2. 准备

- Linux（x86_64 / arm64 等，与 [Release](https://github.com/youfun/caddy-alidns/releases) 架构一致）
- 域名解析在 **阿里云云解析**
- `sudo` 权限
- RAM 子账号的 **AccessKey**（勿把密钥提交到 Git）

---

## 3. 安装

### 3.1 一键安装（推荐）

```bash
curl -fsSL https://raw.githubusercontent.com/youfun/caddy-alidns/main/install_caddy_github.sh | sudo bash
```

脚本行为：

- 有 **Go**：本机 `xcaddy` 编译并安装
- 无 Go：从 **youfun/caddy-alidns** 最新 Release 下载
- 创建用户 `caddy`、`/etc/caddy`、**systemd** 服务并启动

验证：

```bash
caddy version
systemctl is-active caddy
curl -s "http://127.0.0.1"   # 默认提示已安装
```

确认含 alidns 模块（可选）：

```bash
caddy list-modules | grep -i alidns
```

### 3.2 仅下载二进制

从 [Releases](https://github.com/youfun/caddy-alidns/releases) 选择文件：

| 平台 | 文件名 |
|------|--------|
| Linux x86_64 | `caddy-alidns-linux-amd64.tar.gz` |
| Linux ARM64 | `caddy-alidns-linux-arm64.tar.gz` |
| Linux ARMv7 | `caddy-alidns-linux-armv7.tar.gz` |
| macOS (Apple Silicon) | `caddy-alidns-macos-arm64.tar.gz` |
| Windows x64 | `caddy-alidns-windows-amd64.zip` |

Linux 安装示例：

```bash
sha256sum -c caddy-alidns-linux-amd64.tar.gz.sha256
tar -xzf caddy-alidns-linux-amd64.tar.gz
sudo install -m 0755 caddy-linux-amd64 /usr/local/bin/caddy
```

未使用一键脚本时，需自行配置 systemd 或 `caddy run`；可参考脚本中的 `/etc/systemd/system/caddy.service` 片段。

## 4. 配置阿里云密钥

1. [RAM 控制台](https://ram.console.aliyun.com/) 创建子用户，授予目标域名 DNS 解析相关权限（最小权限）。
2. 创建 AccessKey ID / Secret。

在服务器写入环境变量文件：

```bash
sudo tee /etc/caddy/caddy.env <<'EOF'
ALIYUN_ACCESS_KEY_ID=你的AccessKeyId
ALIYUN_ACCESS_KEY_SECRET=你的AccessKeySecret
EOF
sudo chown root:caddy /etc/caddy/caddy.env
sudo chmod 600 /etc/caddy/caddy.env
sudo systemctl restart caddy
```

一键安装生成的服务已包含 `EnvironmentFile=-/etc/caddy/caddy.env`。

---

## 5. 配置 Caddyfile

路径：`/etc/caddy/Caddyfile`。更多示例见 [EXAMPLES.md](../EXAMPLES.md)。

### 5.1 全局 `acme_dns` + 通配符（教程常见写法）

很多教程会在全局块写 `email` 与 `acme_dns alidns`，再为 `*.example.com` 配置站点：

```caddyfile
{
	email you@example.com

	acme_dns alidns {
		access_key_id {env.ALIYUN_ACCESS_KEY_ID}
		access_key_secret {env.ALIYUN_ACCESS_KEY_SECRET}
	}
}

*.example.com {
	# 反代、静态站等
	reverse_proxy 127.0.0.1:8080
}
```

> **安全提示**：请勿在 Caddyfile 里写死 `"你的AccessKey"` / `"Secret"`（教程为演示常这样写，易泄露进 Git 与备份）。请使用上文 [§4 环境变量](#4-配置阿里云密钥) 中的 `{env.ALIYUN_...}`。

### 5.2 单站点 HTTPS + 反代

```caddyfile
{
	email admin@example.com

	acme_dns alidns {
		access_key_id {env.ALIYUN_ACCESS_KEY_ID}
		access_key_secret {env.ALIYUN_ACCESS_KEY_SECRET}
	}
}

app.example.com {
	reverse_proxy 127.0.0.1:3000
}
```

### 5.3 生效配置

```bash
sudo caddy fmt --overwrite /etc/caddy/Caddyfile
sudo caddy validate --config /etc/caddy/Caddyfile
sudo systemctl reload caddy
sudo journalctl -u caddy -f
```

证书与状态数据目录：`/var/lib/caddy`（属主 `caddy`）。

## 6. 日常运维

| 操作 | 命令 |
|------|------|
| 状态 | `sudo systemctl status caddy` |
| 重载配置 | `sudo systemctl reload caddy` |
| 重启 | `sudo systemctl restart caddy` |
| 日志 | `sudo journalctl -u caddy -f` |

临时前台调试：

```bash
export ALIYUN_ACCESS_KEY_ID=...
export ALIYUN_ACCESS_KEY_SECRET=...
sudo -E caddy run --config /etc/caddy/Caddyfile
```

## 7. 升级

1. [Releases](https://github.com/youfun/caddy-alidns/releases) 下载新包  
2. 替换 `/usr/local/bin/caddy`  
3. `sudo systemctl restart caddy`

或重新执行一键安装脚本（保留 `/etc/caddy`）。

---

## 8. 卸载（可选）

```bash
sudo systemctl disable --now caddy
sudo rm /etc/systemd/system/caddy.service
sudo systemctl daemon-reload
sudo rm -f /usr/local/bin/caddy
```

按需删除 `/etc/caddy`、`/var/lib/caddy`。

## 9. 使用常见问题

**证书申请失败**

- 域名是否在阿里云 DNS；AccessKey 能否改该域 TXT 记录  
- `journalctl -u caddy` 查看 ACME / alidns 错误  
- `caddy.env` 权限 `600`，变量名与 Caddyfile 中 `{env.ALIYUN_...}` 一致  

**安装脚本下载 Release 失败**

- 确认仓库已有 [成功的构建与 Release](https://github.com/youfun/caddy-alidns/actions)  
- 或在服务器安装 Go，脚本会改用 xcaddy 本地编译  

**和 Docker 的关系**

- 本仓库面向 **裸机 / systemd**；若要用容器，可把 Release 里的 `caddy` 拷进自建镜像

## 10. 可选：证书同步到阿里云 CDN

若流量走 **阿里云 CDN**，源站（本机 Caddy）上的证书更新后，**CDN 控制台里的证书不会自动跟着变**，需要另行上传 `fullchain.pem` / `privkey.pem` 到各加速域名。

社区示例脚本（与本仓库无隶属关系，使用前请自行审计）：

- [dangjinghao/alicdn_cert_upload](https://github.com/dangjinghao/alicdn_cert_upload) — 依赖 `aliyun` CLI、`jq`，在续期后把证书推到 CDN。

建议：RAM **最小权限**、避免长期用 root 跑、私钥不要通过命令行参数暴露；可在 Caddy **证书续期钩子**（`caddy reload` / `deploy`）里调用该脚本。详见该仓库 README。
