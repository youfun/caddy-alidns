# 编译与发布说明

面向 **维护本仓库** 的同学：如何用 xcaddy 编译、如何跑 CI、如何发 Release。  
**在服务器上安装使用** 请看 [使用说明.md](./使用说明.md)。

---

## 1. 仓库做什么

- 用 [xcaddy](https://github.com/caddyserver/xcaddy) 将官方 Caddy 与插件 **`github.com/caddy-dns/alidns`** 打成单个二进制  
- GitHub Actions 为多平台构建并上传到 **Releases**  
- `install_caddy_github.sh` 供终端用户安装（与本文档受众不同）

参考实现：[openmindw/caddy-cloudflare](https://github.com/openmindw/caddy-cloudflare)

---

## 2. 版本策略

| 项 | 建议 |
|----|------|
| 默认 Caddy 版本 | **v2.10.0**（与 alidns、Go 1.24 CI 已验证） |
| `latest` | 可能拉到 Caddy 2.11+，要求 **Go ≥ 1.25.1**，CI 会失败 |
| 插件 | 固定 `--with github.com/caddy-dns/alidns` |

修改默认版本：`.github/workflows/build-caddy-alidns.yml` 中 `workflow_dispatch` 默认值与 Build 步骤里的 `VER="v2.10.0"` 回退逻辑。

## 3. 本地编译

### 3.1 脚本（推荐）

```bash
git clone https://github.com/youfun/caddy-alidns.git
cd caddy-alidns
./scripts/build-local.sh
CADDY_VERSION=v2.10.0 ./scripts/build-local.sh
```

产物：`dist/caddy`。

### 3.2 手动 xcaddy

```bash
go install github.com/caddyserver/xcaddy/cmd/xcaddy@latest
xcaddy build v2.10.0 --with github.com/caddy-dns/alidns --output ./caddy
./caddy version
```

环境：Go **1.22+**（本机编译 v2.10.0 已用 Go 1.24 验证）。

---

## 4. GitHub Actions

Workflow 文件：`.github/workflows/build-caddy-alidns.yml`

### 4.1 构建矩阵

- linux: amd64, arm64, armv6, armv7  
- darwin: amd64, arm64  
- windows: amd64, arm64  

产物命名：`caddy-alidns-<平台>.tar.gz` 或 `.zip`，附带 `.sha256`。

### 4.2 触发方式

| 触发 | 结果 |
|------|------|
| 推送 tag `v*.*.*` | 正式 Release，tag 名即版本 |
| `workflow_dispatch` + `create_release=true` | 手动 Release（常为 prerelease） |
| 每周日 02:00 UTC | `weekly-YYYYMMDD-HHMM` Release |

### 4.3 用 gh 手动触发

```bash
cd caddy-alidns
gh workflow run "Build Caddy with AliDNS Plugin" \
  -f caddy_version=v2.10.0 \
  -f create_release=true \
  -f manual_tag=latest

gh run list --limit 3
gh run watch <run-id> --exit-status
```

### 4.4 正式发布 tag

```bash
git tag v2.10.0
git push origin v2.10.0
```

Actions 会自动构建并创建 **Release v2.10.0**（非手动 prerelease 命名）。

## 5. 发布检查清单

- [ ] `caddy_version` 使用明确 tag（如 `v2.10.0`），避免误用 `latest`  
- [ ] 8 个 build job 全部成功  
- [ ] Release 附件含各平台包 + sha256  
- [ ] 在 Linux 上 `sha256sum -c` 与 `./caddy version` 抽检  

---

## 6. CI 故障排查

**`go: go.mod requires go >= 1.25.1`**

- 构建时用了 `latest` 或未指定版本，拉到了新版 Caddy。  
- 处理：workflow 输入改为 `v2.10.0`，或把 `actions/setup-go` 升到 1.25.x 并设 `GOTOOLCHAIN=auto`。

**Release job 未跑**

- 需满足：`refs/tags/v*`、定时 `schedule`，或 `workflow_dispatch` 且 `create_release=true`。

**修改仓库 / fork**

- 改 `install_caddy_github.sh` 中 `GITHUB_REPO` 默认值  
- fork 后需在 fork 仓库启用 Actions 并配置 `GITHUB_TOKEN`（默认即可写 Release）
